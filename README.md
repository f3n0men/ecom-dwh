# ecom-dwh

Небольшое хранилище данных для интернет-магазина на MS SQL Server. Три source-таблицы (клиенты, товары, заказы) --> DWH-слой с инкрементальной загрузкой --> витрина топ-5 товаров по городам.

## Структура

```
source layer       klienti, tovari, zakazi          - операционные данные как есть
dwh layer          customer, products, orders       - очищенные данные с суррогатными ключами
datamart layer     dm_top_products_by_city          - агрегат для аналитики
```

Каждая DWH-таблица хранит `dwh_created_at` / `dwh_updated_at` - чтобы всегда было понятно когда строка появилась в хранилище и когда последний раз менялась.

## Запуск

Скрипты выполнять по порядку:

```
01_ddl.sql             - создание таблиц и индексов
02_test_data.sql       - тестовые данные: 50 клиентов, 20 товаров, 100 заказов
03_load_procedures.sql - процедуры загрузки + функция парсинга веса
04_datamart.sql        - процедура расчёта витрины
```

После этого:
```sql
EXEC dbo.usp_run_dwh_load;          - загружает данные из source в dwh
EXEC dbo.usp_build_dm_top_products; - пересчитывает витрину
```

## Загрузка данных

**customer, products** - MERGE с прямым сравнением атрибутов. Источник не хранит `updated_at`, поэтому watermark здесь не работает - сравниваем поля напрямую и обновляем если что-то изменилось.

**orders** - MERGE с watermark по `updated_at`. Таблица заказов растёт постоянно, полный скан каждый день избыточен - берём только строки изменившиеся после предыдущей загрузки.

**Soft-delete у products** - если товар пропал из источника, помечаем `is_deleted = 1` вместо физического удаления. Удалять нельзя: на товар могут ссылаться старые заказы, и мы потеряем историю продаж.

## Витрина

Топ-5 товаров в каждом городе по количеству проданных единиц среди заказов со статусом `PAID` или `SHIPPED`. Используется `DENSE_RANK` — если два товара продались одинаково, оба получают одинаковый ранг и оба остаются в топе, в отличие от `ROW_NUMBER` который выбрал бы одного произвольно.

Витрина пересчитывается полностью при каждом запуске — она небольшая по объёму, зато всегда консистентна.

## Парсинг веса

У товаров поле `weight` на источнике не заполнено, но вес часто указан в описании: «Греча, 800 г.», «Молоко, 1 л.» и т.д. Функция `fn_extract_weight` вытаскивает это значение и записывает в поле при каждой загрузке. Поддерживаемые форматы: г., кг., мл., л.
